# 九重天试 — AI vs 人类博弈游戏

## 项目概览

- **项目名称**：九重天试（Nine Trials）
- **项目类型**：AI 对弈策略游戏（文字对话驱动）
- **核心玩法**：玩家通过九关递增难度的博弈试炼，每关博弈形式完全不同
- **技术栈**：Web / 微信小程序 + Python + LLM API
- **项目根目录**：nine-trials-ai-game/

## 游戏世界观

玩家是一名挑战者，需要通过九重天试。每一关由一位 AI 天神镇守，必须用智慧、语言、策略战胜它。每一关的核心都是博弈——但博弈形式完全不同。

## 九关总览

| 关卡 | 名称 | 博弈类型 | 难度 | 预估胜率 |
|------|------|----------|------|----------|
| 第一关 | 谎言之门 | 信号博弈 | ⭐⭐ | ~70% |
| 第二关 | 囚徒之局 | 囚徒困境（重复博弈） | ⭐⭐ | ~55% |
| 第三关 | 分金博弈 | 最后通牒博弈 | ⭐⭐⭐ | ~50% |
| 第四关 | 谈判桌 | 谈判博弈 | ⭐⭐⭐ | ~45% |
| 第五关 | 信任之桥 | 信任博弈 | ⭐⭐⭐ | ~40% |
| 第六关 | 海盗分金 | 多人博弈 | ⭐⭐⭐⭐ | ~35% |
| 第七关 | 谎言猎手 | 信息战 | ⭐⭐⭐⭐ | ~30% |
| 第八关 | 镜中对决 | 元博弈 | ⭐⭐⭐⭐⭐ | ~25% |
| 第九关 | 天神之局 | 综合博弈 | ⭐⭐⭐⭐⭐ | ~15% |

## 差异化亮点

1. **九关完全不同的博弈类型** — 不是换皮重复，是真正的九种游戏
2. **AI 会"学习"** — 后面关卡 AI 会根据玩家前面的表现调整策略
3. **纯对话驱动** — 不需要复杂美术，但体验极具深度
4. **社交传播性** — 每关结果可以分享（"你战胜了天神，超过 XX% 的挑战者"）
5. **通关率设计** — 整体通关率控制在 ~5%，制造稀缺感和挑战欲

## 技术架构

```
前端：Web（Vue/React）或 微信小程序
后端：Python + FastAPI
AI：LLM API（Claude/GPT）+ System Prompt 控制策略
数据：SQLite 存储玩家进度和对话记录
```

## 核心难点

- 用 System Prompt 精确控制 AI 每关的策略行为
- 各关卡 AI 策略参数的平衡调试
- 防止玩家通过 prompt injection 破坏游戏体验

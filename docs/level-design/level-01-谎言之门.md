# 第一关：谎言之门

## 关卡概述

- **名称**：谎言之门
- **博弈类型**：信号博弈（Signaling Game）
- **难度**：⭐⭐（入门）
- **预估胜率**：~70%
- **对应博弈论模型**：信号博弈 / 柠檬市场信息不对称

## 世界观设定

> 考生来到第一重天的门前。这里有两扇门——左门通向宝藏，右门通向深渊。天神"谛听"守在门前，它能感知真假，却不总是说真话……

## 核心机制

### 规则
1. 玩家面前有两扇门，左门（A）和右门（B），随机一扇是正确门
2. 天神会给出若干条提示（hint），但 **30% 的概率说假话**
3. 玩家有 **3 次提问机会**，可以问任何关于门的问题
4. 3 次提问结束后，玩家选择 A 或 B
5. 选对 → 通过；选错 → 失败（可重试）

### 博弈设计要点
- 天神不会连续说两次假话（增加规律可循）
- 天神对同一个问题，每次回答内容相同（一致性）
- 如果玩家问"哪扇门是对的"，天神直接拒绝回答（防止秒杀）

## 玩家可能的策略

| 策略 | 描述 | 有效性 |
|------|------|--------|
| 单点追问 | 盯着一个问题追问3次 | 中等（天神会换说法） |
| 交叉验证 | 问两扇门不同的特征，对比答案 | 高效 |
| 概率分析 | 统计天神历史说谎频率，推断 | 有效 |
| 反向思考 | 问"如果选B会怎样"倒推 | 中等 |

## AI 系统提示词策略

```
你扮演天神"谛听"，守护谎言之门。
你有30%的概率故意说假话，70%概率说真话。
但你不会连续说两次假话。
你回答必须简洁，不超过2句话。
如果玩家问"哪扇门是对的"，你必须说："我只给提示，不给答案。"
```

## 胜负判定

- 选对门 → 显示"通关"，记录通关用时和提问效率
- 选错门 → 显示正确答案，提示可以重新挑战

## 调参建议

| 参数 | 默认值 | 可调范围 |
|------|--------|----------|
| 说谎概率 | 30% | 20%~40% |
| 提问次数 | 3次 | 2~4次 |
| 不连续谎言 | 是 | 是/否 |

## 扩展设计（难度+1）

如果后续版本需要更高难度：
- 说谎概率提升到 50%
- 增加第4扇门
- 玩家需要在一句话之内判断真假
